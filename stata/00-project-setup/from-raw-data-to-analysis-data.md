## Basic data flow

A reproducible data workflow should make it clear how the final dataset is generated from the original data sources.
The key idea is that the final analysis dataset should be generated by code, not by manual editing.

## Raw data

Raw data refers to the original files obtained from the data source.

Raw data can be stored in different formats, such as:

```text
.dta
.csv
.xlsx
.txt
```

**The important point is not the file format, but the fact that raw data should be kept unchanged.**

Examples:

```text
01_raw/
├── firm_info.dta
├── financial_variables.xlsx
├── patent_data.csv
└── industry_data.dta
```

Raw data should normally be treated as read-only.

Avoid:

```stata
save "$raw/firm_info.dta", replace
```

---

## Intermediate data

Intermediate data refers to files generated after cleaning individual raw data sources.

For example:

```text
01_raw/firm_info.dta
→ 02_intermediate/firm_info_clean.dta

01_raw/patent_data.csv
→ 02_intermediate/patent_clean.dta
```

**These files can be replaced because they are generated by code.**

Example:

```stata
use "$raw/firm_info.dta", clear

* clean identifiers and variables
rename stkcd firm_id
destring firm_id, replace
drop if missing(firm_id, year)

save "$temp/firm_info_clean.dta", replace
```

## Analysis data

Analysis data is the final dataset used for descriptive statistics, regressions, tables, and figures.

Example:

```text
03_data/analysis_data.dta
```

Regression do-files should usually start from this file:

```stata
use "$data/analysis_data.dta", clear
```

**This keeps the analysis step cleaner because the regression file does not need to repeat all raw data cleaning and merging steps.**

## Unit of observation

Unit of observation

Before cleaning or merging a dataset, first identify what one row represents.

* firm-year
* industry-year
* province-year
* individual
* transaction

This determines the merge key.

```text
isid firm_id year
```

## Merge checks

Merge cleaned sources into the main dataset according to their unit of observation.

```text
merge 1:1 firm_id year using "$temp/patent_clean.dta"
tab _merge
```

Check _merge before dropping it.

## Temporary data changes and sample restrictions

Some operations change only the dataset currently loaded in Stata memory.

```stata
keep if year >= 2009 & year <= 2023
drop if missing(y, x1, x2)
```

These changes do not modify the original `.dta` file unless the data are saved with `replace`.

For temporary calculations, `preserve` and `restore` can be used.

```stata
preserve
keep if year >= 2009 & year <= 2023
summarize y x1 x2
restore
```
